This afternoon I encountered my first problem with Visual Studio 2010 –

 

“The "Exec" task needs a command to execute.”

I had no idea what this was. I searched Google the error and found out that it was due to a bug in Visual Studio. It was because there was an empty line in a post-build event I had deleted earlier.

The article I found was referring to Visual Studio 2005 - http://www.kenegozi.com/blog/2007/05/24/the-quotexecquot-task-needs-a-command-to-execute-the-poor-thing.aspx

I’m a bit puzzled as to why a simple bug like this can STILL exist since it was first discovered in 2005 edition – all it needs to do is to use the String.IsNullOrWhiteSpace extension method
